# Lecture 18：以人为本的人工智能

## 这讲在讲什么

这讲关注 AI 系统对人的影响，包括公平性、安全性、隐私、可解释性、偏见、社会影响等。

## 核心概念

| 概念 | 解释 | 和 AI-ISP 的关系 |
|---|---|---|
| Bias | 数据和模型可能对不同群体表现不一致 | 肤色、场景、设备差异 |
| Fairness | 输出质量不能系统性偏向某类数据 | 人像 ISP 很重要 |
| Privacy | 数据采集和训练的隐私风险 | 手机图像数据敏感 |
| Interpretability | 理解模型行为 | failure case 分析 |
| Safety | 避免不可靠输出造成伤害 | 医疗/驾驶/安防图像尤其重要 |

## 为什么它和 AI-ISP 有关系

图像恢复不是纯技术分数。模型可能：

- 把某些肤色处理得更差。
- 在暗光人脸上过度平滑。
- 生成不存在的纹理。
- 对某些手机 sensor 泛化差。
- 在评价指标高时主观画质仍然不好。

因此画质算法也需要责任边界。

## AI-ISP 中的具体问题

1. 人像美化和真实记录之间的边界。
2. 低光增强是否制造虚假细节。
3. 训练数据是否覆盖不同肤色、光源、地区、设备。
4. 模型失败时是否有可解释的 fallback。
5. 端侧处理和云端处理的隐私差异。

## 和本项目的连接

后续做 failure case report 时，不只记录 PSNR，还要记录：

- 偏色。
- 过平滑。
- 假纹理。
- 暗部脏。
- 高光异常。
- 人脸/肤色异常。

这就是把 human-centered AI 落到画质工程。

## 最小带走

1. 画质算法不是只追分。
2. 数据覆盖和失败案例分类很重要。
3. AI-ISP 输出应该忠实、稳定、可解释。
4. 面试时能讲这些，会显得你不只是会跑模型。
